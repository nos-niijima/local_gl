# local_gl ルール実装版 — 仕様書

- 日付: 2026-05-29
- 基準: 既存 `index.html` の実装挙動と物理ボードゲームのルール
- 目的: 既存の「手動コーディネーター」を、役職・行動制限・勝敗を扱う**ルール実装版**にする。

## ルール（確定）

### 役職と陣営
| 役職 | 陣営 | 認識 | 行動制約 | 能力 |
|---|---|---|---|---|
| 航海士 | 人間 | 航海士同士は互いを認識 | 常に操舵＋〇（談話/図書に行けない） | 要職 |
| 乗客 | 人間 | なし | 行先自由 / 必ず〇 | なし |
| ハデス | カロン陣営 | カロンを一方的に認識 | 行先自由(図書推奨) / 必ず〇（×不可） | 両陣営を把握 |
| カロン | カロン陣営 | カロン同士は互いを認識 | 行先も〇×も自由 | 攻撃(談話を出した日のみ)＋妨害(×) |
| 亡霊 | 脱落者の状態 | — | 発言・投票不可 | 毎日1人の行先を談話に上書き(同人連続不可) |

### 人数別（既存 getRoleBreakdown と一致）
5:[航1乗2カ1ハ1] 6:[航2乗2カ1ハ1] 7:[航2乗3カ2ハ0＋亡霊] 8:[航2乗3カ2ハ1＋亡霊] 9:[航2乗4カ2ハ1＋亡霊]

### 1日の流れ（4フェーズ）
①朝の議論＋行先/〇×提出 → ②カロン密談(攻撃指定) → ③昼の行動(公開→撃破判定→操舵〇×集計→図書) → ④夜の追放(投票)

### 得点（既存実装を踏襲＝枚数方式）
- 操舵者数 < 操舵必要 → ×1点（カード非公開）
- 操舵者数 ≥ 操舵必要 → ×が1枚以上なら ×点(=×枚数) / 全員〇なら 〇点(=〇枚数)
- 撃破された操舵者(💀)はカウントしない

### 攻撃/談話/図書（既存実装）
- カロンは「自分が談話を出した日」のみ攻撃可。対象が操舵/図書→撃破、対象が談話→攻撃失敗でカロン自滅。
- 図書: 各自ゲーム中1回。同日複数人→誰も見られずカード消費。
- 亡霊: 入力typeを ghost にし、対象1人の行先を談話に上書き。

### 勝敗
- 人間勝利: 〇が規定数(targetC)到達 or カロン全滅
- カロン勝利: ×が規定数(targetX)到達 or 生存者の過半数がカロン陣営(カロン＋ハデス)（charonFaction*2 ≥ living）
- 判定: 夜の追放後の人数判定を優先、決まらなければポイント到達

## 決定事項（ユーザー確認済み 2026-05-29）
- **本人確認/役職秘匿**: 端末ごとに座席(プレイヤー枠)を localStorage に保持。役職秘匿は**クライアント側のみ**（サーバーは全役職を配信、UIで自分の分だけ表示）。
- **デザイン**: ミニマル・モバイル最優先（大きなタップ領域・余白・明快な配色、ライト/ダーク両対応）。
- **ルール解釈**: 上記でOK。

## データモデル追加（rooms/{id} 配下）
- `settings.ghostEnabled`: bool（亡霊の有無。既定: 7人以上 true）
- `dead`: { p1:true,... }（脱落者）
- `game.winner`: null | "人間" | "カロン"（決着で設定しUIをロック）
- 座席(自分が誰か)は **DBに置かず** localStorage `gl_seat_<room>` に保持。
  座席の埋まり具合は既存 `names` で把握。

## 9要望の対応
1. 座席選択＋自分の役職表示 … トップに「あなた」選択（localStorage）＋自分の役職/説明を表示
2. 役職ごとの操作制限 … 自分の役職で行先/〇×/攻撃をロック（航海士=操舵〇固定、乗客/ハデス=〇固定、カロンのみ攻撃可）
3. 陣営勝利で終了＋勝利陣営表示 … checkWin() で判定→勝利バナー＋ロック
4. デザイン刷新 … ミニマル・モバイル最優先
5. 投票で追放 … 夜フェーズで自動集計（過半数で1人、同数は無し）→ dead 設定
6. 脱落者は亡霊以外は操作不要 … dead かつ !ghost は観戦、dead かつ ghost は上書き操作のみ
7. 役職説明 … 役職ごとの説明文＋ルールヘルプ
8. 他人の役職を見れない … UIは自分の座席の役職のみ表示
9. 亡霊の有無を開始前に設定 … settings.ghostEnabled トグル
