# Ghost Liner（ローカル版）

Netlify 版 Ghost Liner を、外部 Firebase に依存せず **このMac上で完全ローカルに**動かす版です。
同一 Wi-Fi 内のスマホから複数端末で同期して遊べます。クラウドは一切使いません。

## 必要なもの

- Node.js（インストール済み。`node --version` で確認）
- それだけ。`npm install` は不要（Node 標準ライブラリのみで動作）。

## 起動

```sh
cd ghost_liner
node server.js
```

起動すると接続用 URL が表示されます:

```
このMac:    http://localhost:8765
スマホから: http://192.168.x.x:8765   (同じWi-Fi内)
```

- **このMacで遊ぶ**: ブラウザで `http://localhost:8765`
- **スマホで遊ぶ**: Mac と同じ Wi-Fi に繋ぎ、表示された `http://192.168.x.x:8765` を開く

> `start.command` をダブルクリックしても起動できます（Finder から実行）。

### Mac の LAN IP を自分で調べる場合

```sh
ipconfig getifaddr en0   # Wi-Fi。繋がらなければ en1 も試す
```

### ポートを変える

```sh
PORT=3000 node server.js
```

## 遊び方（同期の仕組み）

- 全員が**同じ「room（ルーム）番号」**を開くと同期します。URL の `?room=123456` がルーム番号です。
- 1人が画面右上などでルーム番号を変更/共有 → 他の端末も同じ番号にすると同じ卓に入れます。
- どの端末の操作も即座に全員へ反映されます（元の Firebase 版と同じ挙動）。

## スマホから繋がらないとき（共有Wi-Fi / 外出先）

マンション共用Wi-Fiやカフェ等は「端末分離（クライアントアイソレーション）」で**同じWi-Fiでも端末同士が通信できない**ことがあります。その場合は次のどちらか:

### 方法1: Cloudflareトンネルでネット経由にする（推奨・無料）

LANを経由せずインターネット越しに繋ぐので、分離Wi-Fiでも外出先でも遊べます。

```sh
brew install cloudflared      # 初回だけ（アカウント不要・無料）
./share.command               # サーバー起動＋トンネル公開を一括（ダブルクリックでも可）
```

実行すると `https://〇〇〇.trycloudflare.com` というURLが表示されます。それを各スマホで開くだけ。
- アカウント登録・クレカ・ポート開放、すべて不要。通信量の課金もなし。
- **URLは起動するたびに変わります**（その都度共有）。遊ぶ間はMacを起動したまま（`share.command` がスリープも自動で抑止）。
- 公開URLは推測困難な乱数。データはこのMacの `db.json` に残り、Cloudflareは通すだけ。

### 方法2: テザリング / 自前ルーター（クラウドを一切使いたくない場合）

誰かのスマホのインターネット共有（ホットスポット）か、自前のWi-Fiルーターに全員をつなぐ。
その上で `node server.js` を起動し、表示される `http://<MacのIP>:8765` を各スマホで開く。

## データの保存先

- ルームの状態は `db.json`（このフォルダ）に保存されます。Mac を再起動しても残ります。
- まっさらにしたいときは `db.json` を削除（サーバー停止中に）。

## 構成

| ファイル | 役割 |
| --- | --- |
| `server.js` | ローカル同期サーバー（静的配信 + ロングポーリング購読 + 書き込み + 永続化）。依存ゼロ。 |
| `public/index.html` | アプリ本体。元の Netlify 版のコピー（Firebase 読み込みを shim に差し替えただけ）。 |
| `public/firebase-shim.js` | Firebase API を server.js への通信に翻訳する互換レイヤ。 |
| `start.command` | サーバーだけ起動（ローカルLAN用）。 |
| `share.command` | サーバー＋Cloudflareトンネルを起動（外/分離Wi-Fi用）。 |
| `db.json` | ルーム状態の永続化ファイル（自動生成）。 |

> 同期方式: 当初は SSE を使っていましたが、Cloudflare等のプロキシがSSEをバッファして届かないため、
> どの経路でも通る**ロングポーリング**に変更しています（体感は即時）。

## メモ

- 画面フォントに Google Fonts（Noto Serif JP）を使っています。オフライン環境では
  自動的に代替フォントになります（動作には影響しません）。完全オフラインにしたい場合は
  フォントをダウンロードして `public/` に置き、`index.html` の `fonts.googleapis.com` の
  `<link>` を差し替えてください。
- 身内の LAN 内利用を前提とした作りです（認証・アクセス制御はありません）。
